Vulnerability #2 — CSRF Attack

Cross-Site Request Forgery (CSRF) tricks a logged-in user into submitting a request they didn't intend.

⚠️ Attack Flow

// Malicious site sends this hidden form
<form action="https://bank.com/transfer"
      method="POST">
  <input type="hidden" name="to" value="attacker"/>
  <input type="hidden" name="amount" value="5000"/>
</form>
<script>document.forms[0].submit()</script>

✅ Prevention

// Server generates a random CSRF token
<input type="hidden"
  name="csrf_token"
  value="a3f7x9z2...random..."/>
// Server validates token matches session
// Token is unique per session — cannot forge
