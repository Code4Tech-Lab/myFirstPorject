Threat Landscape & Definitions

Core concepts from the syllabus explained with examples.

Phishing

The practice of sending fraudulent communications that appear to come from a reputable source, typically through email or SMS.

Example: A fake "Password Reset" email from your bank that links to a copycat login page.

Denial-of-Service (DoS)

An attack meant to shut down a machine or network, making it inaccessible to its intended users by flooding it with traffic.

Example: Sending 100,000 requests per second to a web server to crash it.

Spamming

Sending unsolicited messages (usually in bulk) for advertising or distributing malware.

Example: Comment sections filled with links to "get rich quick" schemes.

SQL Injection

Injecting malicious SQL code into input fields to bypass authentication or steal data.

Defense: Using parameterized queries (PreparedStatement) so user input is never executed as code.

Server-Side Security Mastery

💻

Session Hijacking

Attackers steal session cookies to impersonate users.

Defense Strategy

  • Set httpOnly flag (JS cannot read cookie)
  • Set Secure flag (HTTPS only)
  • Implement SameSite=Strict
📂

Path Traversal

Using ../ in URLs to access system files like /etc/passwd.

Defense Strategy

  • Sanitize file paths
  • Use allowlists for file access
  • Run server as non-privileged user
