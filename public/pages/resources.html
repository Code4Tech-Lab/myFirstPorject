🚀 Deployment Options

1

Create a GitHub repository

Go to github.com and create a new repository named cyberaware (or any name). Make it public.

2

Push your code

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/cyberaware.git
git push -u origin main
3

Enable GitHub Pages

In your repo: Settings → Pages → Branch: main → Save. Your site will be live at https://your-username.github.io/cyberaware/ within minutes.

1

Sign up at Netlify

Go to netlify.com and sign up with GitHub. It's free for static sites.

2

Drag & Drop or Connect GitHub

Either drag your /cyberaware folder to the Netlify dashboard, or connect your GitHub repo for auto-deploy on every push.

3

Configure custom domain (optional)

Netlify provides free HTTPS via Let's Encrypt automatically. Add a custom domain in Site Settings → Domain Management.

1

Get a VPS

Use DigitalOcean, Linode, AWS Lightsail, or Oracle Free Tier. Get Ubuntu 22.04 LTS. Note your server IP address.

2

Install Nginx

sudo apt update && sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx
3

Upload files via SFTP (FileZilla)

Open FileZilla. Enter: Host: your server IP, Port: 22, Protocol: SFTP, your credentials. Upload /cyberaware to /var/www/html/.

4

Free HTTPS with Certbot

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d yourdomain.com
# Auto-renews every 90 days

♿ WCAG 2.1 AA Accessibility Checklist

ℹ️ WCAG = Web Content Accessibility Guidelines. Level AA is the standard for most government and enterprise sites.
  • ✅Alt text on all informative images. Decorative images have alt="".
  • ✅Form labels with <label for="id"> or aria-label on every input.
  • ✅Keyboard navigation — all interactive elements reachable via Tab, Enter, Space.
  • ✅Skip link at the top for keyboard users to bypass nav.
  • ✅Focus indicators visible and high-contrast on all interactive elements.
  • ✅Colour contrast ≥ 4.5:1 for normal text, ≥ 3:1 for large text (WCAG AA).
  • ✅Semantic HTML — proper use of <header>, <nav>, <main>, <section>, <footer>.
  • ✅ARIA roles & labels on dynamic content and custom widgets.
  • ✅Live regions (aria-live) for quiz feedback and form errors.
  • ✅Language attribute lang="en" set on <html>.
  • ⚠️Motion — Users with prefers-reduced-motion should have animations removed.
  • ⚠️Responsive text — page must be usable at 200% browser zoom without horizontal scroll.

📁 Full SFTP Guide with FileZilla

ℹ️ SFTP vs FTP: SFTP (SSH File Transfer Protocol) runs over port 22 and encrypts all data. Old FTP sends credentials in plaintext — never use it for production.
1

Install FileZilla Client

Download from filezilla-project.org. It's free and cross-platform.

2

Connect to Your Server

Go to File → Site Manager → New Site. Fill in:

Protocol: SFTP – SSH File Transfer Protocol
Host: your.server.ip.address
Port: 22
Logon Type: Key file  (or Normal with username/password)
User: ubuntu  (or your server username)
Key file: path/to/your_key.pem
3

Upload Your Files

In the FileZilla interface, the left panel is your local machine, the right panel is your server. Navigate to /var/www/html/ on the server and drag over the cyberaware/ folder.

4

Set Permissions

sudo chown -R www-data:www-data /var/www/html/cyberaware
sudo chmod -R 755 /var/www/html/cyberaware
5

Security Note

Never transfer files with sensitive credentials or config files via FTP. Always use SFTP (port 22) or SCP. Disable FTP (port 21) on your server: sudo ufw deny 21.
