How the Web Works

Understanding the client–server model, DNS, and HTTP/HTTPS is the foundation of web security.

💻

1. Browser Sends Request

Your browser (client) sends an HTTP/HTTPS request with headers, method (GET/POST), and data to a domain name.

→
🌐

2. DNS Resolution

The Domain Name System translates example.com into an IP address like 93.184.216.34 so packets can be routed.

→
🖥️

3. Server Responds

The web server processes the request, fetches resources, and responds with HTML, CSS, JS files plus an HTTP status code (200, 404, etc.).

→
🔒

4. HTTPS Encryption

TLS/SSL encrypts the entire communication. The 🔒 padlock means data is encrypted in transit — nobody can sniff your passwords.

📁

Deploying with SFTP & FileZilla

SFTP (SSH File Transfer Protocol) encrypts file uploads unlike plain FTP. Use FileZilla to connect: enter your server's IP, port 22, username, and SSH key. Great for deploying to a VPS/VM running Apache or Nginx. Full SFTP Guide →

Security Concepts Visualized

⚠️ Vulnerable

Weak Input Validation

// ❌ No validation — accepts anything
let email = document.getElementById('email').value;
fetch('/login', { body: JSON.stringify({email}) });

An attacker can submit <script>alert(1)</script> as the email — this is XSS.

✅ Secure

Proper Sanitization

// ✅ Validate with regex before sending
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
if (!emailRegex.test(email)) {
  showError('Invalid email format');
  return; // Reject invalid input
}

Always validate on both client and server. Never trust user input.
